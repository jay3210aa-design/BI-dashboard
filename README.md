# BI 整合儀表板

單一 HTML 檔案，純前端，無需安裝任何後端或套件。  
開啟瀏覽器直接使用，所有設定存在 `localStorage`，重新整理不會遺失。

## 功能總覽

| 頁籤 | 功能 |
|------|------|
| 📈 進度追蹤 | 依規則比對 Output 資料，顯示每日達成率卡片，支援 WIP 即時庫存 |
| 📦 WIP 總覽 | 彙總 WIP 檔，依 Owner / Body / Lead 分群，各工站在製數量一覽 |
| ⚙️ 產能計算 | 計算機台需求（Oven / Loader），管理機台清單與產品參數 |
| 🚫 機台限制 | 為特定機台新增使用限制規則，並支援條件查詢 |
| ⚡ 機台效率 | 上傳機台效率檔，查看稼動率與各機台效率分佈 |

> **規則設定** 入口位於「進度追蹤」頁籤右上角的 ⚙️ 規則設定 按鈕。

---

## 使用方式

1. 以瀏覽器直接開啟 `BI整合儀表板.html`
2. 在「**進度追蹤**」頁籤點擊右上角 **⚙️ 規則設定** 新增追蹤規則
3. 回到「**進度追蹤**」，點擊上傳區匯入 Excel 檔案（可同時選取 Output + WIP）
4. 右上角日期選擇器切換查看日期範圍；按「今日」快速跳回今天

## 各頁籤說明

### 1. 進度追蹤

**上傳資料**
- 點擊上傳框，可同時選多個 Excel 檔（.xlsx / .xls）
- 系統自動識別：含 `pass_qty` 欄位 → Output 檔；含 `ChipMos Lot` / `Oper` → WIP 檔

**摘要列**

| 指標 | 說明 |
|------|------|
| 追蹤項目 | 目前規則總數 |
| 達標（≥85%） | 達成率 ≥ 85% 的規則數 |
| 注意（70–84%） | 達成率 70–84% |
| 落後（<70%） | 達成率未滿 70% |

**規則卡片**
- 每條規則一張卡片，左側色條顯示最差日期狀態（綠/黃/紅/灰）
- WIP 工站標籤可點擊，展開 Lot 明細 Modal
- 日期列可點擊，展開 Output 明細 Modal
- 卡片可拖曳排序，順序自動儲存
- 每張卡片底部有備註文字框，內容即時存檔

**右上角工具列**
- **⚙️ 規則設定**：跳轉至規則設定頁（新增 / 編輯 / 刪除規則）
- **⬇ 匯出 Excel**：輸出欄位：Owner、Lead、Part No.、每日目標、日期、產出(K)、達成率%、狀態、WIP各站、備註

---

### 2. WIP 總覽

**摘要列**：顯示各工站合計在製量（K 顆）與 Hold 量

**篩選功能**
- Owner / Body / Lead 即時篩選
- ChipMos Lot（精確比對）/ Cust Lot（模糊比對）搜尋

**表格功能**
- 點欄位標題排序（數值欄預設降冪，文字欄預設升冪）
- 點任一有數值的儲存格，展開 Lot 明細 Modal

**Lot 明細欄位**：Owner、ChipMos Lot、Cust Lot、Device、Oper、Hold Code、Qty、Entity、MVIN Time、BI 條件

> **BI 條件**：自動從 `Flow` 欄位解析 `TFROM` 對應的括號內容（如溫度條件）

**匯出 Excel（兩個 Sheet）**
- `WIP彙總`：群組彙總表（含合計列）
- `逐筆明細`：含 BI 條件的完整原始資料

---

### 3. 規則設定

從「進度追蹤」頁籤右上角的 **⚙️ 規則設定** 按鈕進入。

| 欄位 | 必填 | 說明 |
|------|------|------|
| Owner | ✅ | 前兩碼做比對（大小寫不區分） |
| Lead | ✅ | 完整比對（大小寫不區分） |
| Part No. | 選填 | 支援萬用字元 `*`（如 `GAA112*`、`*NB25I`） |
| 每日目標量（K） | ✅ | 用於計算達成率的基準，單位 K 顆 |

- 支援行內編輯，修改後立即生效
- 順序可在進度追蹤頁籤拖曳調整，規則設定頁籤同步更新

---

### 4. 產能計算

**機台管理**
- 預設機台系列：AC、AF、AK、AM、AN、AW、AQ、AS
- 每個系列可新增/刪除機台編號（如 AC01、AN05）
- 機台卡片顯示需求台數 vs. 可用台數，超限顯示紅色

**產品參數設定**

| 欄位 | 說明 |
|------|------|
| Oven Type | 選擇機台型號（ANDOC7H2 / ANDOC7H3 / ANDOC7_60 / ANDOD6M / ANDOD5 / ANDOD6 / ANDODQ / H5620） |
| Package | 封裝型號（自由輸入） |
| Owner | 客戶代碼 |
| BIB(set) | 使用幾 set BIB |
| BI Time(hrs) | BI 烘烤時間（小時） |
| Operate Time(hrs) | 操作時間，預設 8 hrs |
| Mask Rate | 有效率（0–1），如 0.95 |
| Socket(ea) | 每 BIB 的 Socket 數量 |
| BIB Type | BIB 型別（G / B / P 等，選填） |

計算欄（自動，灰底）：BIB(ea)、Plan Output(K)、Loader、Oven、Turn、Oven OP、Loader OP

- 可匯出 / 匯入 CSV
- 支援拖曳排序

**機台需求警示**：自動標出超限（紅）/ 正常（綠）/ 閒置（黃）

---

### 5. 機台限制

- 可為特定機台編號新增使用限制（Package、Owner、Part No 可空白表示「不限」）
- Part No 支援萬用字元 `*`
- **機台查詢**：輸入或選取機台編號，查看其所有限制條件
- 可匯出 / 匯入 CSV

---

### 6. 機台效率

- 上傳機台效率 Excel 檔（.xls / .xlsx）
- 摘要卡片：機台總數、運行中、閒置、稼動率、效率優/中/低分佈
- 可依機台系列、運行狀態篩選
- 效率分級：≥80% 優、60–79% 中、<60% 低

## 自動更新功能（Dev）

> 僅 `BI整合儀表板_dev.html` 具備此功能，需使用 **Chrome 或 Edge** 瀏覽器。

### 使用流程

1. 開啟 `BI整合儀表板_dev.html`
2. 點擊右上角 **🔄 自動更新** 按鈕，開啟設定面板
3. 對三個資料檔案分別點「選擇檔案」，授權瀏覽器讀取（**僅需授權一次**）
4. 選擇更新間隔（2 / 4 / 6 / 8 / 12 / 24 小時）
5. 點「▶ 開始自動更新」，系統立即讀取一次並開始倒數
6. 之後只需將最新 Excel 存回**相同路徑**，系統到時間自動讀取

| 項目 | 說明 |
|------|------|
| 授權機制 | File System Access API（Chrome / Edge 支援） |
| 間隔選項 | 2 / 4 / 6 / 8 / 12 / 24 小時，預設 4 小時 |
| 倒數格式 | `hh:mm:ss` |
| 狀態顯示 | 上次更新時間 + 更新了哪些檔案 |
| 頁面重整 | 需重新選擇檔案授權（瀏覽器安全限制） |

### 支援自動更新的檔案

| 類型 | 識別欄位 |
|------|----------|
| Output | `pass_qty` |
| WIP | `ChipMos Lot` / `chipmoslot` |
| 效率檔 | 固定欄位順序（Entity, PartNo, Package...） |

## 資料格式

### Output 檔案（生產產出）

系統以是否含 `pass_qty` 欄識別為 Output 檔。

| 欄位名稱 | 說明 |
|----------|------|
| Date | 日期，支援 Date 物件或 `YYYY/MM/DD` 字串 |
| Owner | 客戶代碼 |
| Lead | 製程前段代號（如 DD2） |
| Product | 產品型號 |
| Pass_qty | 該筆通過數量（顆） |
| Lot_number | Lot 編號（明細用） |
| Oper | 站別 |
| Entity | 機台 |
| BI Time(Hour) | BI 時間 |
| BIB Label | BIB 標籤 |
| BIB Socket | Socket |
| 車規 | 車規標記 |

### WIP 檔案（在製品）

系統以是否含 `ChipMos Lot` 或 `chipmoslot` 欄識別為 WIP 檔（備選：含 `Oper` 欄）。

| 欄位名稱 | 說明 |
|----------|------|
| Owner | 客戶代碼 |
| Body / Body Size | Die 尺寸 |
| Lead | 製程前段代號 |
| Oper | 站別代碼（用於對應 x30–x70） |
| Hold Code | Hold 原因碼（有值即計入 Hold） |
| Qty | 在製數量（顆） |
| ChipMos Lot | 廠內批號 |
| Customer Lot | 客戶批號 |
| Entity | 機台 |
| MVIN Time | 進站時間 |
| TFROM | 來源站別 |
| Flow / Flow(Temperature) | 製程流程字串，用於解析 BI 條件 |

## 計算公式（產能計算）

設：
- `B` = BIB(set)
- `t_bi` = BI Time (hrs)
- `t_op` = Operate Time (hrs)
- `sum` = t_bi + t_op
- `slot` = ANDO 機型 → 48；其他 → 60
- `eff` = 0.92（固定效率）

| 計算欄 | 公式 |
|--------|------|
| BIB(ea) | `B × slot × 0.92` |
| Plan Output(K) | `B × Socket × slot × 0.92 × 24 ÷ sum ÷ 1000 × MaskRate` |
| Loader | `B × 24 ÷ sum ÷ 6`（即除以 24/4） |
| Oven | `B × (t_bi + 3.15) ÷ sum` |
| Turn | `B × 24 ÷ sum` |
| Oven OP | `Turn ÷ 2 ÷ 10` |
| Loader OP | `Loader ÷ 3` |

機台需求 = `ceil(Oven)` 台數（無條件進位）

## 本機儲存（localStorage）

| Key | 內容 |
|-----|------|
| `bi_wip_rules_v1` | 追蹤規則陣列（JSON） |
