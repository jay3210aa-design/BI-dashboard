# BI 整合儀表板

純前端單一 HTML 儀表板，整合 WIP 追蹤、產能規劃、機台效率等功能。  
無後端、無 npm、無 build 流程，拖入 Excel 即可使用。

---

## 功能概覽

| 頁籤 | 說明 |
|------|------|
| **WIP 總覽** | 上傳 WIP Excel，依 Owner / Lead / Body 分組，多條件排序，可匯出 Excel |
| **產能規劃 — 第一階段** | 上傳排程 Excel，看板式工單卡片 |
| **產能規劃 — 第二階段** | 進階排程分析 |
| **產能規劃 — 第三階段：機台效率** | 上傳機台 LOG Excel，即時計算每台機台效率並以顏色標示 |

---

## 使用方式

1. 開啟 `index.html`（直接雙擊或用瀏覽器開啟）
2. 切換至目標頁籤
3. 點擊上傳區域或拖曳 `.xls` / `.xlsx` 檔案
4. 資料自動解析並渲染至儀表板

> 本機開啟即可，不需要伺服器。

## WIP 總覽

### 輸入格式

Excel 第一列為表頭，必須包含以下欄位（順序不拘）：

| 欄位 | 說明 |
|------|------|
| Owner | 工單負責人 |
| Lead | 前導站 |
| Body | 機身站 |
| x30 / x40 / x50 / x60 / x70 | 各站數量 |
| Hold | 暫停數量 |
| 合計 | 總數量 |

### 排序邏輯

點擊任一欄標題可排序；多條件排序：**主鍵 → Owner → Lead → Body**（字母升冪補排）。

### 匯出

點擊「匯出 Excel」可下載目前篩選後的資料。

## 機台效率（第三階段）

### 輸入格式（BI_LUL.xls）

| 欄位 | 說明 |
|------|------|
| Entity | 機台名稱 |
| Oper | 作業名稱 |
| Color | BIB 顏色代碼 |
| Qty | 目前在製數量 |
| inTime | 進站時間（格式：`MM/DD HH:MM`） |
| Status | 機台狀態（`RUNNING` / `IDLE` / 其他） |

### 效率計算公式

```
UPH     = Qty ÷ 已運行小時數
效率 %  = UPH ÷ 5000 × 100
```

> `5000` 為目標 UPH（每小時產出件數）

### 效率色碼

| 效率 | 顏色 | 說明 |
|------|------|------|
| ≥ 80% | 🟢 綠 | 高效率 |
| 60 ~ 79% | 🟡 黃 | 中效率 |
| < 60% | 🔴 紅 | 低效率 |

### 卡片排列邏輯

1. RUNNING 機台優先顯示
2. RUNNING 中：低效率 → 中效率 → 高效率（讓問題機台排最前）
3. 其餘機台依機台名稱字母排序

### 篩選功能

- **機台系列**：依 Entity 前綴自動分組（如 LU24、LU26…）
- **狀態**：全部 / RUNNING / IDLE

---

## 狀態摘要列

| 指標 | 說明 |
|------|------|
| 總機台數 | 本次上傳的所有機台 |
| 運行中 | Status = RUNNING |
| 閒置 | Status ≠ RUNNING |
| 稼動率 | RUNNING ÷ 總機台數 × 100% |
| 高效率 | 效率 ≥ 80% 的機台數 |
| 中效率 | 60% ≤ 效率 < 80% 的機台數 |
| 低效率 | 效率 < 60% 的機台數 |

## 技術架構

| 項目 | 說明 |
|------|------|
| 前端框架 | 純 HTML / CSS / JavaScript（無框架） |
| Excel 解析 | [SheetJS](https://sheetjs.com/) v0.18.5（CDN） |

## 本機開發

無需任何安裝步驟，直接以瀏覽器開啟 `index.html` 即可。

若需要本機 HTTP Server（避免部分瀏覽器的 `file://` 限制）：

```bash
python -m http.server 8080
# 開啟 http://localhost:8080
```
